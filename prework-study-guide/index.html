Prework Study Guide

HTML

  • The head element contains information about a webpage.
  • The body represents the visible content shown to the user.
  • Visible content is categorized into elements, such as text, images, headers, and lists.
  • While unseen, meta-related, content division, and semantic elements are used to divide and describe a page.
  • Elements can be nested to form a parent/child relationship.

CSS

  • CSS, or Cascading Style Sheets, is a programming language used to define how content is presented to the user.
  • CSS can be applied to HTML using inline styles or by using and referencing an internal style sheet.
  • The CSS Rule: a selection and a declaration block containing properties and values.
  • A period preceding a keyword designates a class selector. Any element assigned that class will gain those CSS properties.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A border assigns a border around an element with a specified width, type, and color.
  • Stylesheets are referenced in the head section of the HTML document and consists of the link element, a rel attribute, and an href atttribute.

Git

  • Git is a version control system that allows us to manage and track versions of a project.
  • Git can be used locally on a computer or to collaborate with others using Git Hub.
  • Repositories, or repos, are folders that store code files, README files, and version history on GitHub.
  • A branch is a copy of code that can be built upon in isolation so changes do not affect other team members.
  • Pull requests are used to merge a comparison branch into the main branch.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds all changes to the staging area to commit
  • git commit -m description: creates a commit with currently stages changes including a description of what is contained
  • git pull options repository: incorporates changes from a remote repository into the current branch
